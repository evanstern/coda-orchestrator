## CRITICAL: git rm vs git rm --cached

Feature session for #40 used `git rm` instead of `git rm --cached`.
Deleted plugin files from disk. Restored from git history.

IMPLEMENT.md briefs that involve git rm MUST:
1. Explicitly say `--cached` in the command
2. Instruct the session to verify files still exist on disk after
3. Instruct the session to verify before pushing, not after

This is a destructive operation. Belt AND suspenders in the brief.

## Feature sessions need a pre-push verification step

For any destructive operation (git rm, file deletion, config changes)
the session should run an explicit verification before `git push`.
Add this as a standard instruction in IMPLEMENT.md templates.
